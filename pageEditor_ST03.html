Feedback

Technical Description

To build the Feedback page, I used HTML tags like "header", "nav", "main", and "footer" to give the page a clear structure. The feedback form was placed inside a "section" tag to separate it from other content. I used "form", "label", and "input" elements to collect user data. Labels were linked to their inputs using the "for" and "id" attributes to make the form more user-friendly and accessible. In CSS, I styled the page using properties like "padding", "background-color", and "font-family" to make the layout clean and readable. I used "flexbox" to align the form neatly and keep it responsive on different devices.

Accessibility

I added labels to all form inputs so screen readers can describe the fields correctly. The form also includes "aria-required" where needed. I chose high-contrast colors between text and background to help people with low vision. I tested the form with keyboard navigation to make sure users can fill it out without a mouse.

Team

Technical Description

The Team page was created using standard tags like "header", "section", and "article" to break the content into logical parts. Each team member was placed in their own "div" with a profile image, name using "h3", and a short bio using "p". The layout was controlled using CSS "grid" to organize members into a clean, multi-column layout. CSS properties like "border-radius", "box-shadow", and "hover" effects were used to improve the visual design and make the cards more interactive. I also used "max-width" to keep the content readable across devices.

Accessibility

All images of team members have "alt" text that describes the person, so screen readers can announce their roles. Heading levels were structured in order ("h2", "h3", etc.), and the layout is fully keyboard-accessible. I also avoided flashing or moving elements that could distract or harm users with cognitive sensitivities.

Content Page

Technical Description

This page was built to showcase core content using HTML elements like "main", "article", and "h1" to "h3" for structure. I used "p" for text and "img" for images to support the written content. CSS styling was done using "line-height" for readability, and "margin" and "padding" to give spacing between sections. I also used media queries to ensure the content looks good on both phones and desktops. Key properties like "display: flex" and "text-align" helped keep the layout clean and centered.

Accessibility

For accessibility, I used alt text for all content images and ensured text had enough contrast from the background. I kept the reading order clear and logical. The navigation can be used with a keyboard, and font sizes are large enough for most users to read easily. I also avoided using color alone to show meaning, such as using icons or underlining for links.

Challenges and Lessons Learned

One of the main challenges was getting the layout to work on small screens. At first, elements overlapped or looked too small on mobile. I fixed this by learning how to use "media queries" in CSS and using "flex-direction: column" when the screen size is narrow. Another challenge was fixing form validation errors in the HTML validator. I learned to check for missing labels, incorrect attribute usage, and open tags. These mistakes helped me understand how to write cleaner and more correct HTML. I also learned that testing accessibility early saves time later.

Compliance

I made sure all pages follow the Janet regulations for accessibility, layout, and clean HTML structure. This includes using semantic tags like "nav", "main", and "footer", making sure all images have alt text, and checking for contrast ratios. I validated every page using the W3C tools and avoided using JavaScript or features that break accessibility rules. I worked honestly and made sure the pages are suitable for a wide audience.
