👤 User Profile Page

Technical Description

The User Profile page was created using HTML elements like <form>, <fieldset>, and <input> to capture user details. It includes editable fields, buttons, and basic validation. The layout is responsive using flexbox and grid in CSS.

Accessibility

Form inputs are labeled, tab-order is logical, and all icons have descriptive alt text. ARIA labels were included where needed for screen reader compatibility.

🗺️ Sitemap Page

Technical Description

This page uses semantic HTML to list links to every page in the project. It includes visual icons for categories and uses a two-column grid layout for easy scanning. CSS styling ensures clear structure and responsiveness.

Accessibility

All link texts are descriptive. Navigation is keyboard-friendly and optimized for screen readers with hierarchical headings and labels.

🌡️ Content Page: The Science of Global Warming

Technical Description

This page provides informative content on global warming. It uses structured HTML with sections, headings, paragraphs, and images. I used CSS for clean layout, background styling, and image scaling. It integrates icons, infographics, and fact highlights.

Accessibility

Text is screen reader-friendly, all images include alt attributes, and color contrast meets WCAG 2.1 standards. It also avoids overuse of animations or distractions.

⚠️ Challenges and Lessons Learned

Responsive design was a key challenge, particularly in aligning form elements and text content across screen sizes. I learned how to use CSS grid and media queries effectively. Additionally, validating pages with W3C tools taught me the importance of clean, error-free HTML.

✅ Compliance

Pages are built using semantic HTML5 tags and validated via W3C tools. Accessibility is prioritized following WCAG guidelines. All visuals include alt text, and layouts support various screen readers and devices. No deprecated HTML tags or inaccessible scripts were used.
